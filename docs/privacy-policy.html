Política de Privacidade

Última atualização: 12 de julho de 2026

Em uma frase: o Semantic Tab Grouper lê o conteúdo das abas abertas para entender o assunto e sugerir agrupamentos — e faz isso inteiramente dentro do seu navegador. Nada do que a extensão lê é enviado, vendido ou compartilhado com nós ou com terceiros.

1. Quem é responsável por esta extensão

O Semantic Tab Grouper é desenvolvido e mantido por caliberda.com.br. O código-fonte é aberto e pode ser auditado por qualquer pessoa no repositório oficial: github.com/cesarkali/Semantic-Tab-Grouper.

2. Quais dados a extensão acessa

Para funcionar, a extensão precisa ler, de cada aba aberta:

  • A URL e o título da aba;
  • O texto principal do conteúdo da página, extraído com o Readability.js (o mesmo motor de "modo leitura" usado por navegadores) — menus, anúncios e rodapés são descartados.

Esses dados são usados apenas para calcular do que a página trata e comparar esse assunto com o de outras abas abertas na mesma janela.

A extensão não lê senhas, campos de formulário, cookies, dados de autofill ou qualquer informação digitada pelo usuário — apenas o conteúdo visível da página, do mesmo jeito que o "modo leitura" do navegador faz.

3. Onde e como isso é processado

Todo o processamento acontece localmente, dentro do seu navegador, usando um modelo de linguagem (Xenova/all-MiniLM-L6-v2) rodando via WebAssembly num contexto isolado chamado offscreen document. Não existe backend, servidor ou API própria para onde o conteúdo das páginas seja enviado.

Em outras palavras: se você desconectar o computador da internet depois que o modelo já foi baixado uma vez, a extensão continua funcionando normalmente — porque ela não depende de nenhum servidor para analisar suas abas.

5. Compartilhamento com terceiros

Não vendemos, alugamos ou compartilhamos dados do usuário com terceiros, para publicidade ou qualquer outra finalidade. Não existe telemetria, analytics ou rastreamento de uso embutido na extensão.

6. Serviços de terceiros utilizados

Na primeira vez que a extensão analisa uma aba, ela baixa os pesos do modelo de IA (~25 MB) do CDN público do Hugging Face. Esse download:

  • Não envia texto de nenhuma página, URL visitada ou dado do usuário — é uma requisição de leitura de um arquivo público, igual baixar uma fonte ou uma biblioteca de terceiros;
  • Acontece uma única vez; o navegador guarda o arquivo em cache (Cache Storage) para as próximas vezes;
  • Está sujeito à política de privacidade da própria Hugging Face para a requisição em si (como o IP de origem), fora do nosso controle.

7. Permissões do navegador

A extensão solicita as permissões tabs, tabGroups, scripting, offscreen, storage, alarms e acesso a todos os sites (<all_urls>). Cada uma delas existe só para viabilizar ler, comparar e agrupar abas localmente — nenhuma é usada para coletar dados para fora do navegador. O detalhamento de por que cada permissão é necessária está documentado publicamente no repositório do projeto.

8. Seus controles

  • Desativar o detector a qualquer momento, na tela de configurações;
  • Excluir domínios específicos (bancos, e-mail, intranet etc.) da análise;
  • Escolher o modo semântico, por domínio (sem IA) ou híbrido;
  • Limpar o cache de embeddings com um clique, nas configurações;
  • Desinstalar a extensão a qualquer momento — isso apaga todos os dados armazenados localmente por ela.

9. Retenção e exclusão de dados

Configurações e cache de embeddings ficam salvos localmente até você limpá-los manualmente ou desinstalar a extensão. Não guardamos nenhuma cópia desses dados fora do seu navegador, então não há nada para "solicitar exclusão" além do que já está sob seu controle direto.

10. Segurança

Como nenhum dado de navegação sai do dispositivo, a superfície de risco relevante é o próprio código da extensão. Ele é open source e pode ser auditado por qualquer pessoa, e o build publicado na Chrome Web Store é gerado a partir do mesmo código-fonte disponível no repositório público.

11. Crianças

Esta extensão não é direcionada a crianças e não coleta intencionalmente dados de menores de 13 anos.

12. Alterações nesta política

Se esta política mudar, a data no topo desta página será atualizada. Mudanças relevantes na forma como a extensão trata dados também serão refletidas nas notas de versão do repositório.

13. Contato

Dúvidas, pedidos ou relatos sobre esta política podem ser abertos como uma issue pública no repositório: github.com/cesarkali/Semantic-Tab-Grouper/issues.
